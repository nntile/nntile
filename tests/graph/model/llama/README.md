# Llama Model Tests

Tests for the NNTile C++ Llama model components. Testing is done via safetensors disk files (no Python wrappers).

RMSNorm is a shared module in `nntile::module` (see `include/nntile/module/rms_norm.hh`) and can be reused by other models (GPT, etc.).

## Layout

All tensors use **(hidden_size, seq, batch)** in Fortran (column-major) order. The embedding takes input_ids (seq, batch) and produces (hidden, seq, batch) after transpose. LlamaCausal outputs logits (vocab_size, seq, batch).

## Test Coverage

- **llama_config**: LlamaConfig validation and defaults
- **llama_mlp**: LlamaMLP forward (gated MLP with SiLU)
- **llama_attention**: LlamaAttention forward (sdpa_eager-based) + load/save roundtrip via safetensors
- **llama_decoder**: LlamaDecoder forward (full block)
- **llama_model**: LlamaModel forward (embed + layers + norm)
- **llama_causal**: LlamaCausal forward (model + lm_head)

## Test Data (Safetensors)

Test data is generated by `scripts/generate_llama_test_data.py` and saved to `tests/model/llama/data/`. The script creates random weights, input, output reference, and gradients in NNTile format. CMake runs the script at configure time if Python is available.

To generate manually:
```bash
python scripts/generate_llama_test_data.py --output tests/model/llama/data
```

With PyTorch/transformers for reference output:
```bash
python scripts/generate_llama_test_data.py --output tests/model/llama/data
```

## Save/Load and HF Import/Export

Llama building blocks support:
- **save/load**: NNTile-native safetensors via `Module::save()` / `Module::load()`
- **import_hf/export_hf**: HuggingFace format conversion for `LlamaAttention`, `RMSNorm`, and submodules (`Linear`, `Embedding`, `GatedMlp`)

## Running Tests

```bash
ctest -R tests_model_
```
